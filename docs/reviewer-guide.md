# Reviewer Guide

Recommended review order:

1. Open the SPA home page.
2. Scan the reviewer path and process memo.
3. Review Exercise 1 for platform pricing and billing logic.
4. Review Exercise 2 for workflow artifact and system considerations.
5. Review Exercise 3 for working product proof and technical execution.
6. Open the eVoyage live app and GitHub if deeper technical proof is needed.

## What to evaluate

### Exercise 1

- Does the bundle definition identify related messages clearly?
- Does the charging model use delivery evidence, not only Send API success?
- Is the 2-hour rule handled clearly?
- Are abuse and revenue risks controlled?

### Exercise 2

- Is there a concrete Business Portal / workflow artifact?
- Are API, approval, and UX changes explicit?
- Is behavior over time clear?
- Does the design protect template trust?

### Exercise 3

- Does the web app work?
- Is the product problem clear?
- Is the technical architecture credible?
- Does the project show product ownership and execution ability?
